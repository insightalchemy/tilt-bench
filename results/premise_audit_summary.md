# Premise audit summary (v2 -- deconfounded)

v1 measured order_anomaly as any novel local bigram, but a novel template guarantees every bigram touching it is novel too -- v1's order_anomaly (98.64%) and new_or_rare_template (99.31%) were almost the same 348k rows, not independent evidence. v2 replaces it with `pure_order_anomaly`, gated to rows where every template involved is one the node has ALSO produced normally, so it isolates order novelty from content novelty.

Of 348,460 labeled-anomalous rows, 99.3% still introduce a new-or-rare event template (unchanged from v1, global measure). Once content novelty is controlled for, only 1,829 rows (0.52%) show a *pure* order anomaly -- known templates in an unseen order -- because 346,620 rows (99.5%) were excluded for involving at least one template the node hadn't produced normally (i.e. v1's order signal was indeed almost entirely a content-novelty proxy, confirming the confound). Meanwhile 47.0% show a timing-gap outlier overall, but that rate is 46.8% among the 346,039 rows that already have a new/rare template versus 78.8% among the 2,421 rows that don't -- if anything, timing anomalies are *more* common precisely where content novelty is absent, not less, though that second group is small enough (0.7% of all anomalies) that this should be read as a hint rather than a strong result. Before trusting these z-scores: 0.00% of ALL inter-arrival gaps are exactly zero (median per-node zero-fraction 0.00%, 0.0% of nodes are majority-zero-gap) -- using the microsecond-precision timestamp field rather than the coarser whole-second field means the resolution floor is NOT a practical constraint on this signal, contrary to what BGL's reputation for coarse timestamps might suggest; separately, 0 nodes had a degenerate (zero) MAD (none did, in practice) and would have been routed to the pooled global baseline, while 22,749 of the corpus's ~69k nodes fell back to pooled for having too few normal gaps to build a per-node baseline at all.

**Revised takeaway:** the premise still holds, but on much thinner evidence than v1 suggested. Content novelty (new/rare templates) is the dominant, robust signature of native BGL anomalies, and the apparent 'order' signal in v1 was mostly that same content-novelty signal restated as bigrams -- pure order anomalies (known content, unusual sequence) are vanishingly rare (0.52%). The timing-gap signal doesn't cluster with content novelty the way one might expect; it's comparably or more prevalent among the sliver of anomalies that lack content novelty, but that sliver is too small (2,421 rows) and the fully-isolated timing-only case count (1 row) too thin to claim BGL's native anomalies already exercise timing detection. That remains the gap the controlled injector is meant to fill.
